## 1. 背景

### 現状(変更前)

| シート | 行数 | 内容 |
|---|---|---|
| 前提 | 31 | ファンド/投資/Exit/売上前提 |
| PL | 11 | 売上・粗利・OPEX・営業損益 |
| Exit | 12 | 経路別 企業価値 |
| リターン | 23 | 持分/MOIC/ファンドリターナー判定/逆算 |
| 感度分析 | 20 | EV/Sales × Exit持分 |
| ポートフォリオ | 31 | 10社のべき乗則分布 |
| **合計** | **約110** | **6シート** |

### 何が問題か

1. **PL シートが「会計の言葉」になっている**
   - VC は 営業損益・OPEX を見ない(Exit 評価は売上倍率で決まる)
   - 学生が「会計っぽいから入れた」状態に見える

2. **同じ事実を何度も計算している**
   - 売上CAGR延伸(PL) → Exit売上(Exit) → 回収額(リターン) → 感度分析
   - 4経路で同じ数字に触れているが、本質は「2032年Exit売上を1セルで決める」だけ

3. **ナラティブの核が埋もれている**
   - 一番大事な式は **回収額 = Exit売上 × EV/Sales × Exit持分 ≧ 150億**
   - これがリターン!B12 にひっそり置かれており、見せ方になっていない

4. **ポートフォリオシートの長文説明文がエクセル上で読まれない**
   - 「倍率根拠」列に 100 字を超える文が並ぶ(本来 .md 側で語る話)

## 2. 設計原則(本質 vs 装飾)

### 原則

> **VC財務モデルは「2つの問い」にだけ答えるべき**
>
> ① Solafune 1社で ファンドを返せるか? (= 回収額 ≧ 150億)
> ② 10社合計で net DPI 3.0X 行けるか? (= べき乗則の形が機能するか)
>
> それ以外は「①②への裏付け」か「装飾」のどちらか。

### 仕分け

#### ★本質(残す・むしろ太字化)

| 要素 | 役割 |
|---|---|
| ファンド総額 150億 / 閾値 150億 | ファンドリターナーの定義 |
| 初回チケット 5.5億 / Pre-Val 31億 | エントリー持分=15.1% の入力 |
| 2032年 Exit売上(直接入力 100億) | 回収額の最大ドライバー |
| EV/Sales 倍率 17 | comps の核 |
| Exit持分 12%(希薄化後) | 持分の現実 |
| ファンドリターナー判定 IF 式 | 「○ / ×」が1セルで出る所が一番効く |
| 感度分析(EV/Sales × Exit持分) | 「外れても勝てる幅」の装置 |
| ポートフォリオ要約(★1 / ◎2 / ○3 / ×4) | べき乗則のナラティブ |

#### △装飾(削るか1セルに圧縮)

| 要素 | なぜ装飾か | 推奨処理 |
|---|---|---|
| PL の OPEX(売上×40%) / 営業損益 | VC は黒字化を見ない | 削除 |
| GM% の年次逓増(60→72%) | 売上→Exit評価には不要 | 削除 |
| 売上CAGR延伸(2030→2032 を 9% 延伸) | 「Exit売上を直接入力」で代替可 | 入力1セルに統合 |
| 集中度上限 / リザーブ45:55 / 費用控除後可投資資本 | ファンド運営細部、ピッチ本体に出ない | 注記行に圧縮 |
| 「目標エントリー持分 vs 実エントリー持分 差」 | 補助計算、結論は単一行で十分 | 削除 |
| リターンシート「持分10%/11%/現持分 で必要なExit企業価値」3行 | 感度分析と冗長 | 削除 |
| ポートフォリオシートの長文「倍率根拠」 | エクセルに長文は読まれない | 1〜2語に短縮(根拠は前提パック.md へ) |
| 検算ブロック(リザーブ検算・上位2社シェア等) | 別添に近い | 下部に小さく残す / 削除 |

## 5. 意思決定ポイント

カイトさんが各 D で **A / B / C** のどれか(または「修正」)を選択してください。

### D-1: PL シートを削除する?

- **A: 完全削除** (推奨)
  - 営業損益・OPEX は VC が見ない、削除して問題なし
- **B: 売上行だけ残して PL を 1 行に圧縮**
  - 「PL という単語」を残したい場合
- **C: 現状維持**
  - 削除しない理由がある場合

### D-2: 売上CAGR延伸 (2030→2032) をやめて「2032年Exit売上」を直接入力に変える?

- **A: 直接入力に変更** (推奨)
  - 100 億を直接入力。CAGR の議論は前提パック .md 側で語る
- **B: CAGR延伸を残す**
  - 数式チェーンとしてセットで見せたい場合
- **C: 両方持つ**(片方を隠し計算で)
  - 冗長性は増すが安全

### D-3: Exit シート(独立)を ② ファンドリターナーシートに統合する?

- **A: 統合**(推奨)
  - 「Exit売上 × EV/Sales × Exit持分」は 1 セルで終わる話
- **B: 独立維持**
  - 経路別(IPO/M&A/セカンダリ)を見せたい場合
- **C: 統合するが「経路別」は注記 1 行に**
  - 中間案

### D-4: ポートフォリオシートの長文「倍率根拠」を 1〜2 語に短縮する?

- **A: 短縮(2語以内)**(推奨)
  - 根拠は `01_前提パック.md` で完全に語る
- **B: 短縮(20字程度)**
  - 一目で理解させたい場合
- **C: 現状維持**
  - 長文をエクセルに残す理由がある場合

### D-5: 「集中度上限・リザーブ・費用控除後可投資資本」を前提シート末尾の「注記行」に圧縮する?

- **A: 注記行に圧縮**(推奨)
  - 計算には使わず、見せるためだけに残す
- **B: 完全削除**
  - 「使わない情報は削る」を徹底する場合
- **C: 現状維持**
  - 後述の Q&A で必ず聞かれるため残す場合

### D-6: 各シートの先頭にナラティブ1行を入れる?

- **A: 入れる**(推奨)
  - 「② Solafune単独で回収204億≧閾値150億 ⇒ 成立」など
  - エクセルを開いた瞬間に結論が分かる
- **B: 入れない**
  - 数字だけで語らせる
- **C: 入れるが控えめに**
  - フォント/色で目立たないように

### D-7: 「リターン」シートの逆算3行(持分10%/11%/現で必要なExit企業価値)を削除する?

- **A: 削除**(推奨)
  - 感度分析と情報が冗長
- **B: 残す**
  - 別角度の感度として価値あり
- **C: 感度シートに 1 行統合**
  - 「持分10%で必要なEV=1,500億」など参考表記

## 7. やらないこと(明示)

- **数値前提の変更**: 100億/17倍/12% などの数字自体は変えない(=議論済みの確定値)
- **`01_前提パック.md` の構造変更**: モデル簡素化に伴う **参照側 .md の追記** は別タスク
- **スライド (`03_スライド骨子.md`) の連動更新**: モデル簡素化が確定した後に別途検討
- **comps テーブル / 公的需要プール / risks の再調査**: スコープ外

## 8. リスク / 注意点

1. **数式チェーンの破損リスク**
   - 6 シート間の参照を 4 シートに圧縮するため、リンク切れに注意
   - 実装時に全セルの fully calculated value をバックアップ前後で照合する

2. **PL シート削除の心理的影響**
   - 「PL がないのは手抜きでは?」と Q&A で聞かれる可能性
   - 回答: 「VC評価は売上倍率で決まる。OPEXは Sensitivity に組み込んで Exit売上の精度に集中した」と整理

3. **Excel上のフォーマット**
   - 緑/黄色のセル分け(編集可能=黄、算出=緑)は維持する
   - 既存の「仮・要検証」フラグも保持

4. **不可逆操作の防止**
   - 実装は新ファイル `02_財務モデル.new.xlsx` に書き、確認後にスワップ
   - 既存の `.bak.xlsx` はバックアップとして温存
