## 3. 制作パイプラインと**数値波及ルール（最重要）**

成果物は一方向に依存している。**上流を直すと下流に必ず波及する。**

```
01_前提パック ─→ 02_財務モデル ─→ 03_スライド骨子 ─→ 04_プレゼン台本 ─→ 05_発表スライド
  (市場/競合)      (数値の根拠)       (見せ方)          (話す・守る)        (本番資料)
```

- **数値を変えたら必ず 02 → 03 → 04 → 05 の順に反映する。** 途中を飛ばさない。
- 例：EV/Sales倍率を 17倍→15倍 に下げる → 02のExit/リターン再計算 → 03の該当スライド数字 → 04の台詞 → 05の作図、を全部直して初めて「完了」。
- 数字の不整合は採点軸③④で最も刺さる弱点。**1か所だけ直して放置しない。**

## 4. 規約（全員・AI共通）

- **［仮・要検証］を消すな、潰せ。** 一次ソース未確認の数値は必ず `［仮・要検証］` を付ける。検証して裏が取れたら印を外し、出典を併記する。**勝手に断定しない。**
- **数値には出典を併記。** 「市場のX%」ではなく「誰に・いくらで・何件」のボトムアップで語る（SOM積算の思想）。
- **保守側で置く。** 倍率・成長は保守レンジ（例：EV/Sales 15〜18倍）で置き、上振れは語らない。
- **評価基準①〜⑤を常に意識**（`evals/eval.md`）。スライド/台本は各要素がどの基準に効くかを `outputs/README.md §5` に対応させる。
- **8分（480秒）厳守。** 03骨子と04台本の秒数合計を480秒に同期。1枚＝1メッセージ。
- **レビューの重大度プレフィックス：** `Critical:` / `Important:` / `Nit:` / `FYI:`。無印＝必須。
- **言語：成果物・コメント・コミットは日本語。**
